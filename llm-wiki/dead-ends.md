# Dead Ends

> 已证伪/失败的方向。区分**方向性证伪**（不要再试）和**实现性缺陷**（改进实现后可重试）。

---

## 归因层级

每条 dead-end 必须明确归因层级：

- **方向性错误** — 方向本身不可行 → 不再尝试
- **实现性缺陷** — 实现有问题但方向可能可行 → 改进实现后可重试
- **条件性失效** — 在特定条件下失败，换条件可能有效 → 加约束后可重试

---

## 写入模板

```
### DN. 一句话主题

**证据**：版本号、backtest ID、关键指标对比，链接到 performance-matrix.md 对应行。

**为什么**：根因分析。

**注意**：是否有"换条件可重试"的边界。
```
