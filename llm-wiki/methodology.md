> 纯方法论。不存数据和案例，所有具体数字引用 wiki 其他文件。

## 一、分析前必做：确认条件

**每次分析必须先记录并标注以下条件，不标条件的结论 = 无效结论：**

| 条件 | 为什么重要 |
|------|-----------|
| **初始资金** | 不同量级行为差异巨大（仓位上限、张数限制、分散度） |
| **测试区间** | 牛市/熊市/震荡市结论可能完全相反 |
| **样本量** | <50 笔统计意义极低 |

## 二、Results JSON 分析

### 关键指标

```python
stats = result["backtest"]["statistics"]
net_profit = stats["Net Profit"]       # 绝对收益
sharpe = stats["Sharpe Ratio"]         # 风险调整收益
drawdown = stats["Drawdown"]           # 最大回撤
total_orders = stats["Total Orders"]   # 交易频次
win_rate = stats["Win Rate"]           # 胜率
pl_ratio = stats["Profit-Loss Ratio"]  # 盈亏比
```

### 常见陷阱

| 陷阱 | 表现 | 应对 |
|------|------|------|
| Sharpe 虚高 | 短周期(3 个月)Sharpe>10 | 检查区间长度，<1 年不可信 |
| PSR vs NP 混淆 | QC 界面 PSR 栏看起来像 Net Profit | 从 result JSON 取真值 |
| DD 假象 | 修正日期区间后 DD 剧变 | 确保覆盖至少一个完整熊市 |
| 交易数骤降 | 新参数导致交易从 200→30 笔 | 交易数<基线 50% 时需警惕 |

### 跨版本对比规则

1. **只对比相同条件下的回测** — 同资金、同区间
2. **分年看** — 全周期均值掩盖单年崩溃
3. **关注 DD 变化而非 NP 变化** — NP 在牛市被放大，DD 更稳健
4. **参数组合不可加** — 单参数各 +30% ≠ 组合 +60%

## 三、Trades CSV 分析

### 基础：用 trades 不用 orders

`trade_analysis.py` 的 `load_and_pair_orders()` 配对买卖计算真实盈亏。trades CSV(round-trips)优于 orders CSV — orders 包含未成交、部分成交等噪音。

### 维度价值排序

按信息量从高到低，决定分析优先级：

| 优先级 | 维度 | 发现方式 | 为什么有价值 |
|--------|------|---------|------------|
| **P0** | Top/Bottom 标的 | ticker groupby | 利润集中度揭示策略本质 |
| **P0** | 持仓天数分桶 | holding_days | 赢/输单的时间结构差异 |
| **P1** | 年-月交叉 | year×month pivot | 定位具体崩溃月份 |
| **P1** | 入场日内涨幅 | gap + intraday_return | 跨版本常见的稳健正向因子 |
| **P2** | RSI/BB 分桶 | 需配合市场环境 | 环境依赖 |
| **P2** | VIX 区间 | vix_level 分桶 | 高 VIX 区策略表现往往显著下降 |
| **P3** | 行业板块 | sector groupby | 主题集中度，但样本量通常不足做板块结论 |
| **低** | 入场时间/星期几 | 分桶 | 通常无显著规律 |

### 关键分析模式

**模式 A — 利润集中度**
```
Top N 标的利润 / 总利润 → 集中度
```
集中度 >80%：策略是"事件彩票机"，过滤器改不了命运。
集中度 <50%：策略有泛化能力，可考虑选股优化。

**模式 B — 赢家/输家画像对比**
`discover.py` 自动做 t-test。关注 p<0.05 的特征。
**陷阱：统计显著 ≠ 实际有用** — 过滤器在剔除"平均差"交易的同时可能杀死极端尾部暴利。

**模式 C — What-If 沙盘推演**
```
过滤条件 X → 规避亏损 M 笔($Y) + 误杀盈利 P 笔($Q) → 净影响 $(Y-Q)
```
**看绝对金额，不只看笔数。** 过滤掉 100 笔小亏($300/笔)但误杀 1 笔大赢($50K)是灾难。

**模式 D — 环境分段对比**
按 SPY>MA200 / SPY<MA200 拆分交易集，分别统计胜率和 EV。判断是否需要 Regime 的最直接方法。

### 不要做的分析

| 分析 | 为什么避免 |
|------|-----------|
| 行业胜率差距判断 | 样本量通常不足，差异可能是随机噪音 |
| 基于 <30 笔样本的结论 | 高方差策略小样本不可靠 |
| 单季度 Sharpe 对比 | 时间窗口偏差 |

## 四、结论方法论

### 结论三层级

| 层级 | 要求 | 标注方式 |
|------|------|---------|
| **方向性结论** | 多版本、多区间、大样本一致 | "已确认: ..." |
| **环境依赖结论** | 标明在哪个条件下成立 | "在 X 条件下: ..." (资金/区间/市场环境) |
| **假设性发现** | 标明样本量和置信度 | "假设(N 笔, p=X): ..." |

### 辩证失败归因

每个"失败"必须区分归因层级（详见 dead-ends.md 顶部说明）：

- **方向性错误** — 方向本身不可行 → 不再尝试
- **实现性缺陷** — 实现有问题但方向可能可行 → 改进实现后可重试
- **条件性失效** — 在特定条件下失败，换条件可能有效 → 加约束后可重试

### 对比分析检查清单

进行任何对比分析前：

- [ ] 两组使用相同初始资金？
- [ ] 两组覆盖相同测试区间？
- [ ] 样本量是否都 >50 笔？
- [ ] 是否覆盖至少一个牛熊周期？
- [ ] 结论是否标注了所有前提条件？
- [ ] 是否查了 dead-ends.md 确认没有重复探索？

## 五、Kill Criteria 模板

**动手前必须定义**，防止沉没成本让失败策略苟活。每个新大版本（vX.0）启动时填入 `validation-plan.md`：

| 指标 | 默认阈值 | 说明 |
|------|---------|------|
| IS Sharpe | < 0.8 → 否决 | 低于无风险利率+风险溢价的最低水准 |
| IS Max DD | > 40% → 否决 | 超过此值实盘几乎无法坚持 |
| 相对主题基准年化超额 | < 5% → 否决 | 证明有 stock selection alpha，不是 beta 复制 |
| 任何子区间回撤 | > 50% → 否决 | 单一市场事件能打爆策略 |
| 交易数骤降 | < 基线 50% → 警惕 | 信号过滤可能杀死尾部赢家 |
| **单票 PnL 集中度** | **> 50% → 否决；30-50% → 警示** | **alpha 若高度依赖单只票，就是侥幸而非能力** |

**单票集中度规则原理**：高集中度可能掩盖"过关的 Sharpe/DD"实则"alpha 靠一只彩票"的脆弱性。

- **> 50%**：否决 —— 策略本质是押注一只票，无泛化能力
- **30-50%**：警示 —— 需取证追问"去掉该票后是否仍有超额"
- **< 30%**：通过 —— alpha 在多票上合理分布

每个版本回测报告必须显式报告 Top 1 / Top 3 / Top 5 ticker PnL 贡献占比。

阈值可按策略类型调整，但必须**动手前写死**，不得在结果出来后松口。

## 六、风险指标检查清单

**不只看 Sharpe**。每次回测报告必须同时展示：

| 指标 | 公式/含义 | 为什么重要 |
|------|----------|-----------|
| **Sharpe** | (年化超额收益) / 年化波动 | 标准风险调整收益，但对动量策略不够 |
| **Calmar** | 年化收益 / 最大回撤 | 动量策略更关键 —— DD 是实盘生死线 |
| **Sortino** | (年化超额) / 下行波动 | 不惩罚向上波动 |
| **Max DD Duration** | 最大回撤持续天数 | 深度不是唯一维度，时长同样影响持有意愿 |
| **Tail Ratio** | 95% 分位收益 / \|5% 分位收益\| | 尾部对称性；动量策略理想 >1 |
| **Hit Rate** | 胜率 | 低胜率+高盈亏比是动量/彩票策略正常形态 |
| **Turnover** | 年化换手率 | 判断交易成本侵蚀比例 |
| **Beta vs 主题基准** | 对主题 ETF 的回归 beta | 若 beta≈1 且 alpha≈0 → 只是主题复刻 |
