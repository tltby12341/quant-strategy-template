过夜自主迭代模式 — 用户离开后 Claude 全自主运行。

## 启动前确认

1. 读取 skill 文档: `strategy_research/skills/StrategyForge/SKILL.md`
2. 读取工作流: `.agent/workflows/strategy-forge.md`
3. 检查当前基准:
   ```bash
   python3 strategy_research/skills/StrategyForge/evaluate.py show-baseline
   ```
4. 确认用户提供了：
   - 当前回测 ID（或正在运行的回测）
   - 对比版本（如 v2.62 vs v2.63）
   - 优化方向（可选，如果没有则从数据分析中确定）

## 工作模式

**全自主闭环：监控 → 分析 → 迭代 → 监控 → ...**

1. **监控回测**：从 0% 开始轮询 API 检查进度，不刷浏览器
2. **回测完成后**：立即下载结果（分页获取全部 orders）、运行 evaluate 评估
3. **数据驱动决策**：根据分析确定下一步方向，不停下来问用户
4. **表现差早停**：不等跑完，果断停止并换方向
5. **一次一个变量**：用数据判断该变量是否是根因
6. **记录每个实验**：写入 experiments.tsv，无论成功失败

## 关键决策原则

- 改了变量但指标没变 → 该变量不是根因，立即换方向
- 追溯历史版本找真正的分歧点，不在最新版本上反复微调
- 不同版本各有优势时，尝试组合它们的长处

## 铁律（继承 StrategyForge）

1. 所有数字来自数据，不估算
2. 一次只改一个变量（参数扫描除外）
3. 表现差立即停止
4. 不放松验收标准
5. F 系列命名
6. 回测检查从 0% 开始
7. 不刷新 QC 浏览器，API only

## 用户回来时

汇报总结：每个实验的结果表格 + 关键发现 + 当前最优版本状态
